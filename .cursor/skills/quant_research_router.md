# Skill: 券商金工研报复现路由器

## 适用场景

当用户输入一篇券商金工研报（标题、摘要、或全文）时，使用本 skill 识别研报类型并路由到对应的子 skill 进行量化研究复现。

## 研报类型识别规则

根据研报中的**关键词**和**研究主题**，将研报归类到以下类别，并调用对应的 skill 文件：

### 类别一：基本面选股

**触发关键词**：F-Score、FFScore、Piotroski、基本面、选股法则、价值投资、低PB、ROA、ROE、财务指标、财务质量、现金流选股、大师系列、基本面量化

**对应 Skill**：`.cursor/skills/skill_fundamental_stock_selection.md`

**QuantsPlaybook 参考目录**：`QuantsPlaybook/A-量化基本面/`

**匹配的已有研究**：
| 研报主题 | 参考实现 |
|---------|---------|
| Piotroski F-Score / 财务质量评分 | `A-量化基本面/华泰FFScore/FFScore.ipynb` |
| 大师选股法则 / 现金流价值投资 | `A-量化基本面/申万大师系列十三/` |
| 企业生命周期 | `B-因子构建类/企业生命周期/` |

## 路由流程

当用户提供研报时，按以下步骤操作：

### Step 1：提取研报元信息
- 标题、作者/券商、发布日期
- 核心研究主题、关键方法论
- 使用的数据源和回测区间

### Step 2：关键词匹配与分类
- 用上述关键词表匹配研报类别和子类
- 如果研报跨多个类别（如"基于因子的择时"），优先匹配**核心方法论**对应的类别
- 如果无法精确匹配，选择最接近的子类

### Step 3：读取对应 Skill
- 根据匹配结果，读取对应的 skill 文件
- 同时读取 `.cursor/skills/skill_toolkit_reference.md` 获取工具函数参考

### Step 4：参考已有实现
- 在 QuantsPlaybook 对应目录中找到最接近的参考实现
- 分析参考实现的代码结构和工作流
- 基于参考实现和研报内容，制定复现方案

### Step 5：执行复现
- 按照 skill 中定义的标准化工作流执行
- 使用 hugos_toolkit 和 SignalMaker 中的工具函数
- 输出标准化的分析报告

## 无法匹配时的处理

如果研报主题在 QuantsPlaybook 中没有直接对应的实现：
1. 选择最接近的类别 skill
2. 参考该类别的通用工作流模板
3. 从零开始实现研报中的方法论
4. 复用 hugos_toolkit / SignalMaker 中的通用工具
